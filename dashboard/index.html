// REALM INTELLIGENCE — DIVISION REFERENCE
J_CLAW oversees 6 divisions, each led by a commander who runs autonomous agents. Each agent produces a packet (JSON report). The division orchestrator synthesizes all packets into a digest. The daily briefing (22:00 or on-demand) compiles all division digests for J_Claw. Escalations bypass the briefing — they are pushed directly to the alert system.
TRADING // SEREN
Iron Codex · Cycle-based market strategy system
Mission: Monitor financial markets, build and test strategies, execute trades, and optimize returns over time.
  • market-scan (2h)Scans live market data for entry signals, volatility shifts, and anomalies across multiple assets.
  • virtual-trader (18:00)Executes simulated paper trades based on the active strategy signals from market-scan — tracks PnL, win rate, and R-multiples.
  • backtester (18:05)Evaluates strategy candidates against historical data and selects the top-performing strategy for the next trading cycle.
  • trading-report (18:10)Synthesizes daily trading performance — win rate, P&L, strategy review, and recommendations.
  • Agent Network (09:05 M–F)Multi-agent trading cycle: Strategy Builder builds the playbook, Backtester validates it, Trader executes, Trading Coach reviews the session.
SEREN synthesizes all trading data and escalates any breach of risk thresholds (drawdown, win rate floor) directly to J_Claw.
OPPORTUNITY // VAEL
Dawnhunt · Lead intelligence and pipeline management
Mission: Scout job leads, contracts, and funding opportunities across the web and surface the best ones for review.
  • job-intake (3h)Scrapes configured job sources, deduplicates listings, and ingests new leads into the pipeline.
  • hard-filter (auto)Applies strict criteria (keywords, salary, location, role) to classify leads as Tier A, B, or C.
  • funding-finder (14:00)Identifies grants, contracts, remote roles, and non-traditional income streams beyond standard job boards.
VAEL tracks pipeline health (sources online, leads/day, pending reviews) and alerts when the review queue goes stale.
DEV AUTOMATION // KAELEN
Veilkeepers · Codebase health and automation
Mission: Maintain codebase quality, automate documentation, and keep the development environment clean and healthy.
  • repo-monitor (3h)Scans git repositories for uncommitted changes, test failures, and dependency drift.
  • debug-agent (event)Responds to error events triggered via Telegram — analyzes logs, traces root causes, and proposes fixes.
  • refactor-scan (10:00)Identifies code smells, dead code, complexity hotspots, and duplication across the codebase.
  • doc-update (13:00)Reviews recent code changes and updates inline documentation, docstrings, and README files.
  • artifact-manager (03:00)Cleans stale build outputs, orphaned packages, large temp files, and expired artifacts.
  • dev-digest (15:00)Synthesizes all dev health metrics into a daily report with prioritized action items.
KAELEN tracks the repo health score and escalates high-priority refactor flags or security issues to J_Claw.
PERSONAL // LYRIN
Thornward · Human performance monitoring
Mission: Track physical health, mental performance, and burnout risk to keep the operator running at peak capacity.
  • health-logger (18:00)Prompts a daily health check-in and logs responses — sleep, energy, mood, focus, physical activity.
  • perf-correlation (20:00)Correlates health metrics with trading performance and productivity output over rolling windows.
  • burnout-monitor (21:00)Analyzes stress indicators across all divisions and computes a composite burnout risk score.
  • personal-digest (21:30)Synthesizes all personal division health data into a daily report with wellness recommendations.
LYRIN monitors the human behind the realm — burnout alerts are escalated immediately and bypass the standard briefing cycle.
OP-SEC // ZETH
Null Breach · Digital security and threat monitoring
Mission: Maintain airtight digital security posture, monitor for breaches, and ensure all credentials and systems are locked down.
  • device-posture (08:00)Audits device security settings — firewall, disk encryption, OS updates, open ports, and remote access.
  • threat-surface (19:00)Maps the current attack surface — exposed services, public accounts, credentials, and entry points.
  • security-scan (11:00)Scans the codebase and npm/pip dependencies for known CVEs and security vulnerabilities.
  • cred-audit (15:00)Reviews API keys, secrets, and credentials for rotation compliance and exposure risk.
  • privacy-scan (16:00)Checks for data exposure, PII leaks, third-party tracking, and excessive data collection.
  • breach-check (14:00)Monitors known breach databases (HaveIBeenPwned, etc.) for compromised credentials.
  • opsec-digest (16:30)Daily security summary with overall threat level, top priority fixes, and compliance status.
  • mobile-audit-review (23:00)Reviews the daily mobile session audit log — validates authentication events, detects anomalous access patterns, and flags suspicious mobile activity.
ZETH operates in silence. Any high-severity finding (breach detected, critical CVE, credential exposure) escalates to J_Claw without delay.
LYKEON FORGE // LYKE
The Lykeon Forge · Creative media production pipeline
Mission: Manage all creative media production — images, sprites, audio, video, and asset delivery through the ComfyUI pipeline.
  • image-generateCalls the ComfyUI REST API to generate images from structured prompt templates per asset type.
  • sprite-generateGenerates FE-style chibi and bust character portraits for commanders, enemies, and UI elements.
  • video-generateVideo generation pipeline — placeholder until video model is active, scaffolded for future use.
  • graphic-designGenerates UI elements, card borders, overlays, and decorative graphics for the game interface.
  • prompt-craftBuilds and refines generation prompts per asset type, commander, and art direction guidelines.
  • style-checkValidates generated assets against the art direction palette using PIL dominant color analysis.
  • image-reviewQA agent — detects generation artifacts, near-black/white failures, and solid-color errors via PIL.
  • audio-testValidates audio files for clipping, silence, abnormal RMS levels using scipy/pydub analysis.
  • video-reviewReviews video for generation failures, codec issues, and frame count validation via ffprobe.
  • storyboard-composeConverts theater animation queue events (skill_complete, rank_up) into visual shot lists for generation.
  • continuity-checkValidates visual consistency across all tiers of a character's evolution — same palette, same pose language.
  • asset-catalogIndexes every generated asset by type, commander, tier, and approval status in state/asset-catalog.json.
  • asset-deliverRoutes approved assets to the correct game directories under mobile/assets/ and updates catalog entries.
  • production-digestThe Forge Report — synthesizes full pipeline status, asset counts, pending QA, and delivery metrics for J_Claw.
LYKE monitors the production pipeline. Any blocked pipeline (ComfyUI down, QA backlog >10) escalates to J_Claw with delivery ETAs.
